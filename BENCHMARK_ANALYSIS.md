# Smore 벤치마킹 분석 및 개선 계획

**작성 일자**: 2024-12-19  
**목적**: Smore의 응답 데이터 도출 UI/UX를 벤치마킹하여 개선하되, 기존 기능을 보존

---

## 📊 Smore의 주요 특징

### 1. 상단 통계 카드
- 참여 (Participants)
- 완료율 (Completion Rate)
- 평균 세션시간 (Average Session Time)
- 링크 클릭 수 (Link Clicks)
- 전환율 (CTR)
- 공유버튼 클릭 수 (Share Button Clicks)

### 2. 탭 네비게이션
- 질문별 인사이트 (Insights by Question)
- 응답별 데이터 (Response Data)
- 기간별 인사이트 (Insights by Period)
- 질문별 이탈률 (Question Drop-off Rate)

### 3. 질문별 인사이트
- 각 질문을 카드 형태로 표시
- 질문 타입에 따라 차트 (Pie, Bar) 또는 리스트 표시
- 질문별 필터/검색 기능
- 응답률 표시

### 4. 응답별 데이터
- 테이블 형태로 개별 응답 데이터 표시
- 체크박스로 다중 선택 가능
- 컬럼별 정렬 기능
- 페이지네이션 (91 / 전체 91)
- 전체 다운로드 버튼
- 검색 필터 추가 기능

### 5. 검색 및 필터
- 질문별 검색 필터 추가
- 날짜 범위 필터
- 응답자별 필터

## ✅ 개선 계획 (기존 기능 보존)

### 1. SurveyResults.jsx 개선

#### 1.1 상단 통계 카드 추가
**추가할 기능:**
- 참여 수 (총 응답 수)
- 완료율 (완료된 응답 / 시작된 응답)
- 평균 세션시간 (응답 시작 ~ 제출 시간)
- 링크 클릭 수 (추후 구현 가능)
- 전환율 (추후 구현 가능)
- 공유버튼 클릭 수 (추후 구현 가능)

**기존 기능 보존:**
- ✅ 기존 통계 데이터 유지
- ✅ 기존 API 호출 유지
- ✅ 기존 데이터 구조 유지

#### 1.2 탭 네비게이션 추가

- 질문별 이탈률 (추후 구현)

**기존 기능 보존:**
- ✅ 기존 질문별 차트 표시 유지

## ⚠️ 주의사항

### 1. 기존 기능 보존
- 모든 기존 기능은 유지해야 합니다.
- 기존 API 호출은 변경하지 않습니다.
- 기존 데이터 구조는 변경하지 않습니다.

### 2. 점진적 개선
- 기존 기능을 먼저 유지한 상태에서 새 기능을 추가합니다.
- 새 기능은 기존 기능과 공존합니다.
- 사용자가 기존 기능을 사용할 수 있어야 합니다.

### 3. 테스트
- 모든 기존 기능이 정상 작동하는지 테스트합니다.
- 새 기능이 기존 기능에 영향을 주지 않는지 확인합니다.
- 데이터 구조 변경 시 하위 호환성을 유지합니다.
